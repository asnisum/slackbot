# Slack 채널 업무 분석

`~/.slackbot/`에 저장된 데이터를 읽어서 팀/개인/프로젝트별 업무를 분석합니다.
**Slack API 호출 없음** — 로컬 저장 데이터만 사용합니다.

## 입력

사용자가 제공하는 인자: `$ARGUMENTS`

- 첫 번째 인자: 채널 ID (필수)
- 두 번째 인자 (선택): 기간
  - `이번주`: 이번 주 월~오늘
  - `지난주`: 지난 주 월~금
  - `이번달`: 이번 달 1일~오늘
  - `YYYY-MM-DD~YYYY-MM-DD`: 특정 기간
  - 미입력 시: 최근 7일
- 세 번째 인자 (선택): 분석 유형
  - `팀`: 팀 업무 현황만
  - `개인`: 개인별 현황만
  - `프로젝트`: 프로젝트별 현황만
  - `전체`: 모든 분석 (기본값)

## 실행 순서

### Step 1: 입력 파싱

`$ARGUMENTS`에서 채널 ID, 기간, 분석 유형을 추출합니다.

### Step 2: 데이터 확인

`~/.slackbot/{CHANNEL_ID}/daily/` 디렉토리에서 해당 기간의 데이터를 확인합니다.

- 각 날짜 디렉토리의 `metadata.json` 존재 여부 확인
- 데이터가 있는 날짜 수 vs 전체 기간 날짜 수 비교

**데이터 부족 시:**
```
⚠️ 분석 기간 {N}일 중 {M}일만 데이터가 있습니다.
더 정확한 분석을 위해 `/slackbot:backfill {CHANNEL_ID} {시작일} {종료일}` 을 먼저 실행하세요.
```
- 데이터가 전혀 없으면 안내 후 종료
- 데이터가 일부라도 있으면 가용 데이터로 분석 진행

### Step 3: 데이터 로딩

1. 각 날짜의 `daily/{YYYY-MM-DD}/metadata.json` 읽기
2. 각 날짜의 `daily/{YYYY-MM-DD}/README.md` (요약) 읽기
3. `metadata.json`의 `thread_ids`로 스레드 `{thread_ts}/README.md`도 읽기
4. 데이터가 많으면 (15일 이상) 요약(`README.md`)만 사용
5. 데이터가 적으면 (14일 이하) 원본(`conversation.md`)까지 참고

### Step 4: 분석

#### 팀 업무 현황
- 기간 내 진행 중인 업무 목록
- 주요 하이라이트 (결정사항, 마일스톤)
- 블로커 및 이슈
- 커뮤니케이션 패턴 (활발한 날, 조용한 날, 토론이 많은 주제)

#### 개인별 현황
- 참여 빈도 (메시지 수, 스레드 참여)
- 주요 업무/담당 영역
- 기여 내용 요약
- 진행 중인 액션 아이템

#### 프로젝트별 현황
- 대화에서 식별된 프로젝트/주제 분류
- 각 프로젝트 참여자
- 진행 상황 요약
- 다음 단계 (Next Steps)

### Step 5: 결과 출력

```markdown
## 📊 업무 분석 리포트

**채널**: #{channel_name}
**기간**: {시작일} ~ {종료일}
**데이터**: {N}일 분석 (메시지 {M}개, 스레드 {K}개)

---

### 팀 업무 현황

#### 진행 중 업무
1. {업무 내용} — 담당: @이름
2. ...

#### 주간 하이라이트
- {하이라이트}

#### 블로커/이슈
- {블로커}

---

### 개인별 현황

#### @이름1
- **참여**: 메시지 {N}개, 스레드 {M}개
- **주요 업무**: {업무 설명}
- **진행 중**: {액션 아이템}

#### @이름2
...

---

### 프로젝트별 현황

#### {프로젝트/주제명}
- **참여자**: @이름1, @이름2
- **상태**: {진행 상황}
- **다음 단계**: {Next Steps}
```

### Step 6: 결과 저장

- 경로: `~/.slackbot/{CHANNEL_ID}/analysis/{오늘날짜 YYYY-MM-DD}/README.md`
- 저장 완료 후 파일 경로를 사용자에게 알려줍니다.

## 에러 처리

- 채널 ID 미입력: "채널 ID를 입력해주세요. 형식: /slackbot:analysis {CHANNEL_ID} [기간] [분석유형]"
- 데이터 없음: "해당 채널의 수집 데이터가 없습니다. `/slackbot:daily` 또는 `/slackbot:backfill`을 먼저 실행해주세요."
- 기간 형식 오류: "기간 형식이 올바르지 않습니다. 이번주, 지난주, 이번달, YYYY-MM-DD~YYYY-MM-DD 중 하나를 입력해주세요."
