# Slack 채널 업무 분석

사용자가 팀/개인/프로젝트별 업무 분석을 요청하면 이 스킬이 자동으로 트리거됩니다.
`~/.slackbot/`에 저장된 데이터를 읽어서 분석합니다 (Slack API 호출 없음).

## 트리거 조건

- "업무 분석해줘"
- "누가 뭐 했어"
- "팀 현황 정리"
- "이번주 뭐 했는지 정리"
- "개인별 현황"
- "프로젝트별 진행 상황"

## 워크플로우

### 1. 입력 파싱

사용자 메시지에서 채널 ID, 기간, 분석 유형을 추출합니다.

**채널 ID**: 메시지에서 채널 ID 또는 URL을 찾습니다. 없으면 사용자에게 물어봅니다.

**기간 결정**:
- "이번주" → 이번 주 월~오늘
- "지난주" → 지난 주 월~금
- "이번달" → 이번 달 1일~오늘
- 기간 미언급 → 최근 7일

**분석 유형**:
- "팀", "팀 현황" → 팀
- "개인", "누가 뭐" → 개인
- "프로젝트" → 프로젝트
- 미언급 → 전체

### 2. 데이터 확인

`~/.slackbot/{CHANNEL_ID}/daily/` 에서 해당 기간 데이터 존재 여부를 확인합니다.

데이터가 부족하면:
```
⚠️ 분석 기간 {N}일 중 {M}일만 데이터가 있습니다.
더 정확한 분석을 위해 `/slackbot:backfill`을 먼저 실행하세요.
```

### 3. 데이터 로딩 및 분석

각 날짜의 `README.md`(요약)와 스레드 `README.md`를 읽어서 분석합니다.

**3가지 분석 섹션:**

| 섹션 | 내용 |
|------|------|
| 팀 업무 현황 | 진행 중 업무, 주간 하이라이트, 블로커, 커뮤니케이션 패턴 |
| 개인별 현황 | 참여 빈도, 주요 업무, 기여 내용, 진행 중 항목 |
| 프로젝트별 현황 | 주제 분류, 참여자, 진행 상황, 다음 단계 |

### 4. 결과 출력 및 저장

분석 결과를 출력하고 `~/.slackbot/{CHANNEL_ID}/analysis/{오늘날짜}/README.md`에 저장합니다.

## 주의사항

- Slack API를 호출하지 않습니다. 로컬 데이터만 사용합니다.
- 데이터가 없으면 `/slackbot:backfill` 또는 `/slackbot:daily`를 안내합니다.
- 결과는 항상 한국어로 출력합니다.
