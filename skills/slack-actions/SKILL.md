# Slack 스레드 액션 아이템 추출

사용자가 Slack 스레드 URL을 제공하거나 스레드 분석을 요청하면 이 스킬이 자동으로 트리거됩니다.

## 워크플로우

### 1. URL 파싱

Slack URL에서 channel_id와 thread_ts를 추출합니다.

**URL 형식**: `https://{workspace}.slack.com/archives/{CHANNEL_ID}/p{TIMESTAMP}`

파싱 규칙:
- `CHANNEL_ID`: `/archives/` 뒤의 경로 세그먼트 (예: `C0123456789`)
- `thread_ts`: `p` 뒤 숫자에서 **뒤에서 6자리 앞에 `.`을 삽입** (예: `p1234567890123456` → `1234567890.123456`)

### 2. 대화 로드

`mcp__slack__slack_get_thread_replies` 도구를 호출하여 스레드 전체 대화를 가져옵니다.

```
channel: {CHANNEL_ID}
thread_ts: {thread_ts}
```

### 3. 참여자 이름 확인

대화에 참여한 각 사용자의 `user` ID로 `mcp__slack__slack_get_user_profile`을 호출하여 실제 이름(display_name 또는 real_name)을 확인합니다.

- 동일한 사용자 ID는 한 번만 조회합니다.
- 조회한 이름을 메시지의 user ID와 매핑합니다.

### 4. 대화 분석

가져온 대화 내용을 분석하여 다음을 추출합니다:

- **참여자**: 대화에 참여한 사람들
- **논의 주제**: 스레드에서 다룬 핵심 주제 요약
- **결정 사항**: 대화 중 합의되거나 결정된 내용
- **액션 아이템**: 누가 무엇을 언제까지 해야 하는지

### 5. 결과 출력

다음 형식으로 결과를 출력합니다:

```markdown
## 📋 스레드 요약

**채널**: #{channel_name}
**참여자**: {참여자 목록}
**날짜**: {스레드 시작 날짜}

### 논의 내용
{주요 논의 내용 요약 - 3~5문장}

### 결정 사항
1. {결정 사항 1}
2. {결정 사항 2}

### 액션 아이템
| # | 담당자 | 할 일 | 기한 | 우선순위 |
|---|--------|-------|------|----------|
| 1 | @이름 | 구체적인 할 일 | YYYY-MM-DD 또는 미정 | 높음/중간/낮음 |
```

### 6. 결과 저장

결과를 `~/.slackbot/{channel-id}/{thread-ts}/` 디렉토리에 저장합니다.

#### 6-1. 원본 대화록 저장 (`conversation.md`)

참여자 이름이 매핑된 원본 대화를 시간순으로 정리하여 저장합니다.

```markdown
# 원본 대화록

**채널**: #{channel_name}
**스레드 시작**: {YYYY-MM-DD HH:MM}

---

**[YYYY-MM-DD HH:MM] 이름(Name)**
메시지 내용 (멘션은 @이름 형태로 변환)

---
```

- 각 메시지의 타임스탬프를 `YYYY-MM-DD HH:MM` (KST) 형식으로 변환합니다.
- `<@U12345>` 형태의 멘션을 `@이름` 으로 치환합니다.
- 메시지 사이를 `---`로 구분합니다.

#### 6-2. 분석 결과 저장 (`README.md`)

4~5단계에서 생성한 요약 및 액션 아이템을 저장합니다.

- `Write` 도구를 사용하여 파일을 생성합니다.
- 이미 파일이 존재하면 덮어씁니다.
- 저장 후 두 파일의 경로를 사용자에게 알려줍니다.

## 주의사항

- 액션 아이템이 명시적으로 언급되지 않은 경우, 대화 맥락에서 암묵적인 할 일을 추론합니다.
- 기한이 명시되지 않은 경우 "미정"으로 표시합니다.
- 우선순위는 대화의 긴급도와 맥락을 기반으로 판단합니다.
- 한국어와 영어가 혼합된 대화를 모두 처리할 수 있어야 합니다.
- 결과는 항상 한국어로 출력합니다.
